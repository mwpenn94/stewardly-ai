ENGINE DASHBOARD ROOT CAUSE ANALYSIS
=====================================

BUG 1: Chart shows no data / tiny values
- heData from API is an ARRAY of YearlySnapshot objects (30 items)
- Code does: setHeResults(heData.map((d, i) => ({ ...d, name, color })))
- Since d is an array, spreading it creates: { 0: snap0, 1: snap1, ..., name, color }
- Then code reads: r.snapshots || [] which is UNDEFINED
- FIX: Store as { snapshots: d, name, color } instead of { ...d, name, color }

BUG 2: Company key mismatch in dropdown
- Router has: wealthbridge, nwm, massmutual, wirehouse, ria, communitybd, diy, donothing, bestoverall
- Engine has: wealthbridge, captivemutual, wirehouse, ria, communitybd, diy, donothing, bestoverall
- "nwm" and "massmutual" don't exist - should be "captivemutual"
- FIX: Update router enum + dashboard dropdown to use correct keys

BUG 3: BIE results mapping
- BIE.simulate() returns BIEYearResult[] (array directly)
- Dashboard reads: bieResults.yearResults || []
- FIX: Pass bieResults directly to IncomeStreamBreakdown

BUG 4: Monte Carlo data shape
- uweMonteCarlo returns MonteCarloPercentile[] from types
- MonteCarloFan expects: { year, p10, p25, p50, p75, p90 }
- Need to verify the actual return shape matches

BUG 5: Missing features
- No quick-load presets (should have: New Associate, Experienced Pro, Director, MD, RVP, etc.)
- No BIE dedicated section with income breakdown
- No v7 context (company descriptions, strategy info, product details)
- No responsive mobile layout
- Strategy cards are cramped and hard to use

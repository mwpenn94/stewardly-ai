GHL Integration Spec v5.0 - Key Findings
==========================================

CUSTOM FIELDS ON CONTACTS:
- calculatorCompletedDate (Date, required, single)
- calculatorLastRunDate (Date, required, single)
- planType (Dropdown: basic|growth|premium|custom, required)
- totalValue30yr (Number, required)
- roi30yr (Text, required - ratio format like "52:1")
- strategyRecommended (Dropdown: conservative-dividend|balanced-growth|aggressive-equity|real-estate-blend|tax-optimized-hybrid)
- affiliateTrackA (Text, optional)
- affiliateTrackB (Text, optional)
- bizIncomeProjection (Number, optional)
- savingsRate (Number, required, 0.01-1.00)
- returnRate (Number, required, decimal)
- planShareUrl (Text, conditional)
- calculatorRunCount (Number, required)

API ENDPOINTS:
- v1: rest.gohighlevel.com/v1/contacts/ (POST create, PUT update)
- v2: services.leadconnectorhq.com/contacts/ (POST create, PUT update)

AUTH:
- v1: Bearer {GHL_API_KEY} static
- v2: OAuth2 access token, 24h expiry, refresh via POST /oauth/token

UPSERT LOGIC:
1. Lookup by email (GET /contacts/lookup?email={email})
2. If found: PUT update, increment calculatorRunCount, update calculatorLastRunDate
3. If not found: POST create, set calculatorCompletedDate, calculatorRunCount=1
4. Race condition: distributed lock on email, 30s TTL

AUTOMATION TRIGGERS:
1. Calculator Completed → tag "calculator-completed" → 3-Email Nurture
2. Plan Shared → tag "plan-shared" → Notify advisor + 48hr follow-up
3. Strategy Comparison → calculatorRunCount >= 2 within 7 days → comparison email
4. High ROI Plan → roi30yr > 50 → priority pipeline + senior advisor

ERROR HANDLING:
- 200/201: success
- 400: log, no retry
- 401: refresh token (v2) or rotate key (v1)
- 422: log, no retry
- 429: exponential backoff 1s,2s,4s,8s,16s max 5
- 500+: retry max 3, then DLQ

WEBHOOK HEADERS:
- Authorization: Bearer {token}
- Content-Type: application/json
- Version: 2021-07-28 (v2 only)
- X-Stewardly-Source: wealthbridge-calculator-v1
- X-Idempotency-Key: {calculationId}-{timestamp}
